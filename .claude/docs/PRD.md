# BNI 富禮一對一預約平台 - 產品需求文件 (PRD)

> 由 `/concept` 維護

---

## 專案概述

### 專案目標

解決 BNI 富禮分會成員之間「一對一（One-to-One）約訪難以敲定時間」的問題。

BNI 分會的引薦運作建立在成員彼此深度了解之上，一對一是核心活動，且次數需回報 PALMS。目前約訪流程依賴 LINE 私訊來回拉扯：「你什麼時候有空？」「我看一下」「那天不行改別天」……協調成本高，導致一對一頻率下降。

平台要把這個流程壓縮成：**打開網頁 → 看到對方可約的時段 → 送出預約 → 對方一鍵確認**。

### 核心價值

1. **消除來回協調**：空檔資訊透明化，預約一次到位
2. **資料不過期**：以「每週固定模式」為基底（設定一次、長期有效），輔以具體時段登記與例外調整，避免手動時段表過期失效的經典死法
3. **不只約時間，還解決「約誰」**：一對一矩陣讓成員一眼看到這季還沒約過誰

---

## 目標用戶

### 用戶畫像

- BNI 富禮分會成員（單一分會，約 20–50 人）
- 多為中小企業主、業務、專業人士；行程忙碌、變動大
- 重度 LINE 使用者；不一定有使用數位行事曆的習慣
- 3C 熟練度落差大 → 介面必須極簡、手機優先

### 使用場景

1. **設定空檔（低頻）**：成員初次加入時設定每週固定可約模式（例：週二下午、週五早上）；行程有變時隨時修改
2. **臨時調整（中頻）**：某週固定時段有事 → 把該週該時段挖掉；某天臨時有空 → 加開一次性時段
3. **發起預約（高頻）**：想約某位成員 → 打開對方頁面 → 看到可約時段 → 選一個送出，可附訊息（例如見面地點提議）
4. **回應預約（高頻）**：收到 LINE 通知 → 點連結 → 確認／提議改期／婉拒
5. **改期或取消**：任一方都可對已成立的預約發起改期或取消，另一方收到通知
6. **找下一個約訪對象**：打開一對一矩陣 → 看到這季還沒約過的成員 → 直接發起預約
7. **完成記錄**：一對一結束後打卡，矩陣更新，方便回報 PALMS

## 功能需求

### 核心功能 (MVP)

- [ ] **成員與登入**
  - LINE Login 登入（免密碼，同時綁定 LINE 身分供推播）
  - 成員基本資料：姓名、公司、專業別、頭像
  - 成員名單僅限分會成員可見（邀請制加入）
- [ ] **空檔管理（混合制，所有內容可隨時修改）**
  - 每週固定模式：設定每週固定可約時段（例：週二 14:00–17:00）
  - 一次性時段：登記特定日期的額外空檔
  - 例外挖除：將某週的固定時段標記為不可約
  - 所有空檔設定可隨時新增、修改、刪除
- [ ] **預約流程（請求 → 確認制）**
  - 瀏覽成員的可約時段（固定模式 + 一次性 − 例外 − 已被預約）
  - 送出預約請求，可附留言（地點、主題）
  - 對方回應：確認／提議其他時段／婉拒
  - 已成立預約可由任一方改期或取消，雙方皆收到通知
- [ ] **LINE 通知**（透過 LINE 官方帳號 Messaging API 推播）
  - 收到新預約請求
  - 預約被確認／改期／取消
  - 約訪前一天提醒
- [ ] **一對一矩陣**
  - 依季度顯示「我跟誰約過／還沒約過」
  - 一對一完成後打卡記錄（任一方標記完成）
  - 從矩陣直接發起預約

### 次要功能

- [ ] 配對建議：主動提示「你這季還沒跟 XXX 約過」
- [ ] 個人統計：本季一對一次數、分會排行
- [ ] PALMS 回報輔助：匯出本週完成的一對一清單
- [ ] 管理員後台：成員邀請與停用、分會設定

### 未來考慮

- Google Calendar 雙向同步（進階成員選用）
- 多分會支援（其他 BNI 分會可自行開設）
- 約訪地點建議（成員常用地點資料庫）

## 非功能需求

### 效能要求

- 單一分會規模（≤ 50 人），流量極小；一般雲端免費方案即可支撐
- 手機開啟到可操作 < 3 秒

### 安全要求

- 所有資料僅分會成員登入後可見；成員名單、聯絡方式、行程不對外公開
- 程式碼開源（public repo），但任何成員資料、金鑰一律走環境變數與資料庫，絕不進版控
- LINE channel secret / access token 存於 `.env`，提供 `.env.example`

### 相容性

- 手機優先（成員主要從 LINE 點連結開啟，in-app browser 必須正常運作）
- 支援桌面瀏覽器

### 通知用量限制

- LINE Messaging API 免費方案每月 200 則推播；以 40 人分會、每人每月 4 場一對一估算，每月推播約 500–800 則，超出免費額度
- 對策：MVP 期間合併通知（一場預約全流程 ≤ 3 則）、每日提醒合併為一則；若仍不足，升級輕中用量方案（約 NT$800/月，由分會負擔）或以站內通知為主、LINE 只推關鍵事件
